# AI Pulse Briefing Generator - 面试演讲稿

---

## Slide 1: 封面页
*   **标题:** AI Pulse Briefing Generator: 资深编辑的自动化工作流
*   **副标题:** Agentic AI 在非结构化信息处理中的应用与落地

**(演讲稿)**

“各位面试官，大家好。我是 [您的姓名]，今天我将向大家展示一个我独立完成的 AI Demo 项目：**AI Pulse Briefing Generator**。这个项目的核心目标，是利用 Agentic AI 的能力，自动化一位资深 AI 科技编辑的日常工作，实现高信噪比的每日新闻简报生成与推送。这个 Demo 不仅展示了 AI 核心逻辑，更重要的是，它体现了从产品构思到工程化落地的完整闭环。”

## Slide 2: 痛点与机会
*   **标题:** 信息过载与低信噪比：AI 时代的新挑战
*   **内容:**
    1.  **痛点:** 每日海量 AI 新闻（模型、论文、融资），人工筛选效率低下，关键信息易被淹没。
    2.  **目标:** 市场需要一个 **高信噪比、专业筛选、定时触达** 的 AI 行业简报。
    3.  **机会:** 利用 LLM 的理解和推理能力，将资深编辑的经验转化为自动化 Agent。

**(演讲稿)**

“我们首先来看一个行业痛点：**信息过载**。每天，全球发布的 AI 新闻数以千计，但真正有价值的，比如 **模型发布、重大融资、政策监管**，可能只有寥寥几条。传统方法依赖人工编辑或简单的关键词过滤，效率低且容易遗漏。我们的目标，就是解决这个 **低信噪比** 的问题。我们看到了一个巨大的机会：利用 LLM 强大的文本理解和推理能力，将一位拥有十年经验的资深编辑的判断力，转化为一个可重复、可扩展的自动化 Agent。”

---

## Slide 3: 产品核心价值
*   **标题:** 核心价值：从“信息流”到“决策点”
*   **内容:**
    1.  **专业筛选:** 自动识别并剔除软文、旧闻，聚焦 **模型发布、重大融资、政策监管、开源突破**。
    2.  **结构化输出:** 严格遵循专业简报格式（Markdown），便于阅读和二次处理。
    3.  **自动化交付:** 实现每日定时、无感知的微信推送（工程化落地）。

**(演讲稿)**

“AI Pulse Generator 的核心价值体现在三个方面：**专业筛选**，我们不再是简单地聚合信息，而是像专业编辑一样，剔除那些不重要的营销软文，只保留对行业有战略影响的动态。**结构化输出**，我们严格按照预设的 Markdown 模板输出，确保内容简洁、专业，可以直接被用户消费。最后是 **自动化交付**，通过工程化手段，确保简报每日定时、稳定地推送到用户手中，实现真正的产品闭环。”

## Slide 4: 技术架构总览
*   **标题:** Agentic Workflow：复杂任务的拆解与控制
*   **内容:**
    1.  **架构:** 采用 **RAG (Retrieval-Augmented Generation) + Agentic Workflow** 模式。
    2.  **流程图:** 绘制一个清晰的 4 步流程图：**数据检索** → **过滤与排名** → **内容生成** → **自动化推送**。
    3.  **核心理念:** 将一个复杂的非结构化任务，拆解为多个可控、可调试的 LLM 步骤。

**(演讲稿)**

“这是项目的技术架构总览。我们没有将整个任务交给 LLM 一次性完成，而是采用了 **Agentic Workflow** 的设计模式。我们将‘生成简报’这个复杂的任务，拆解成了四个独立的、可控的步骤。每一个步骤都对应一次 LLM 的调用或一个工具的执行。这种设计极大地提高了系统的 **准确性、可调试性和稳定性**。例如，如果过滤步骤出错，我们只需要调整该步骤的 Prompt，而不会影响后续的生成逻辑。”

---

## Slide 5: 关键技术点 (I) - 噪音过滤与结构化
*   **标题:** Prompt Engineering：强制 LLM 输出可控结果
*   **内容:**
    1.  **挑战:** 如何让 LLM 准确判断新闻价值并输出程序可读的结果？
    2.  **解决方案:** 使用 **结构化输出 (JSON)** 约束 LLM Call 1 (Filtering)。
    3.  **示例:** 展示 Filtering Prompt 的关键指令（如“剔除超过 48 小时的旧闻”、“输出必须是 JSON 格式”）。

**(演讲稿)**

“在 **噪音过滤** 这一步，我们面临的挑战是如何将 LLM 从一个‘创意写作工具’变成一个‘可靠的数据处理器’。我们的解决方案是使用 **Prompt Engineering** 中的 **结构化输出** 技巧。我们强制 LLM 必须返回一个 **JSON 格式** 的结果，其中包含每条新闻的保留/剔除状态和原因。这不仅确保了输出结果能被后续的 Python 脚本轻松解析，更重要的是，它迫使 LLM 按照我们预设的逻辑（如时间、内容类型）进行判断，极大地提高了过滤的准确性。”

---

## Slide 6: 关键技术点 (II) - 排名与推理
*   **标题:** 模拟专家决策：LLM 的推理与排序能力
*   **内容:**
    1.  **挑战:** 如何模拟资深编辑的“重要性”判断？
    2.  **解决方案:** 使用 **Few-shot Prompting** 和 **Chain-of-Thought** 提示词，指导 LLM 根据预设的优先级（模型 > 芯片 > 融资 > 政策）进行评分和排名。
    3.  **结果:** 确保最重要的 3 条新闻（Top Stories）被准确识别。

**(演讲稿)**

“**排名与推理** 是最能体现 LLM 价值的一步。我们希望简报的排序是基于 **战略价值**，而不是简单的关键词频率。我们通过 **Few-shot Prompting**，向 LLM 提供了几个‘资深编辑’的判断案例，并结合 **Chain-of-Thought** 提示词，要求 LLM 在给出排名之前，先分析新闻对行业的影响力。例如，我们设定了‘模型和芯片的突破’优先级高于‘政策和融资’。这样，LLM 就能模拟专家的判断，确保像摩尔线程发布新架构这样的新闻能被准确地排在 **‘头条关注’** 的位置。”

---

## Slide 7: 工程化落地 (I) - Serverless 部署
*   **标题:** 自动化与弹性：Serverless 部署方案
*   **内容:**
    1.  **部署环境:** 采用 **Serverless Function** (如 AWS Lambda / 腾讯云 SCF)。
    2.  **优势:** 免运维、按需付费、高可靠性，完美适配每日定时任务。
    3.  **调度:** 使用 **定时触发器** (Cron Job) 每日固定时间（如 8:00 AM）启动 Function。

**(演讲稿)**

“再好的 AI 逻辑也需要可靠的工程化落地。对于这种每日定时的任务，我们选择了 **Serverless 架构**。它具有三大优势：**免运维**，我们不需要管理服务器；**按需付费**，只有在每天早上 8 点运行时才产生费用，成本极低；**高可靠性**，云服务商的定时触发器保证了任务的稳定执行。这体现了我们对现代云计算架构的理解和成本控制的意识。”

---

## Slide 8: 工程化落地 (II) - 微信集成
*   **标题:** 解耦与触达：通过 Webhook 实现微信推送
*   **内容:**
    1.  **挑战:** 如何将 Function 的输出推送到微信？
    2.  **解决方案:** 使用 **Webhook** (例如 Server酱 或 企业微信应用) 作为中间件。
    3.  **优势:** 简报生成逻辑与推送逻辑完全解耦，增强系统健壮性。

**(演讲稿)**

“在最终的交付环节，我们通过 **Webhook** 实现了与微信的集成。我们没有直接去处理复杂的微信 API，而是将 Serverless Function 的输出发送到一个标准化的 Webhook 接口。这种设计实现了 **逻辑解耦**：简报生成逻辑和推送渠道是完全独立的。这意味着将来如果我们要增加邮件或 Slack 推送，只需要增加一个新的 Webhook 目标，而不需要修改核心的 AI 生成代码，极大地增强了系统的 **可扩展性**。”
